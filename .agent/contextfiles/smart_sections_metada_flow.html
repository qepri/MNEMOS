2. Data Structure Transformation

How a text string "Gravity" becomes a Node ID in the database.

sequenceDiagram participant S as Summarizer participant DB as DocumentSection Table participant U as Unifier Task participant G as Concept Table (Graph) participant H as HyperEdge Table S->>DB: Save Section 1 (metadata: {concepts: ["Gravity", "Physics"]}) Note right of DB: User sees Summary immediately loop Async Background Process U->>DB: Read new Section Metadata U->>G: Check if Concept "Gravity" exists? alt Concept Exists G-->>U: Return UUID (e.g., 550e8400...) else New Concept U->>G: Create Node "Gravity" G-->>U: Return New UUID end U->>H: Create Semantic Link (HyperEdge) Note right of H: "Section 1 mentions Gravity" end
Key Takeaways:
  • Phase 1 (Done): Fast, parallel summarization stored in `document_sections` with JSON metadata. Great for RAG filtering.
  • Phase 2 (Proposed): An asynchronous "Unifier" task that reads that JSON and converts it into formal Graph Nodes (`concepts`) and Edges (`hyper_edges`).
  • This keeps valid/fast summarization while slowly building the high-quality Knowledge Graph in the background.
